# 地库车辆定位系统路演 PPT 框架设计

## 1. 路演定位

- 目标场景：课程答辩、项目路演、技术展示、成果汇报。
- 核心听众：评委、导师、技术负责人、潜在落地合作方。
- 推荐时长：8-10 分钟。
- 叙事主线：地下车库找车痛点 -> 车牌定位方案 -> 端到端技术闭环 -> 训练与部署效果 -> 后续落地空间。
- 视觉基调：16:9 宽屏，强层级标题，深蓝/青绿智能交通科技感，图表和系统流程优先，少堆文字。

## 2. 一句话定位

地库车辆定位系统通过多路视频接入、YOLO Pose 车牌角点检测、OCR/LPRNet 识别、SQLite 轨迹入库和 Web 可视化查询，实现“输入车牌号，快速定位车辆最后出现摄像头和时间”的完整边缘 AI 应用。

## 4. 每页内容策略

### 第 1-3 页：先让人知道项目有用

- 封面不要塞太多技术词，强调“地库寻车”和“车牌定位”。
- 背景页用“地下停车场空间复杂、人工找车效率低、摄像头数据没有形成可查询轨迹”三点铺垫。
- 方案总览页用一条路径讲清楚：摄像头看到车辆 -> 模型识别车牌 -> 写入轨迹库 -> 用户查询位置。

### 第 4-7 页：证明实现逻辑扎实

- 系统架构页使用层级式框图，不展开代码细节。
- 端到端逻辑页突出每一步的输入、处理和输出。
- 推理后端页明确 PC 后端用于开发联调，RDK 后端用于边缘部署。
- 线程页重点解释“采集不推理、推理只处理最新帧、HTTP 只读快照或查数据库”。

### 第 8-10 页：展示效果和数据支撑

- Web 效果页用页面截图/拟态界面展示最终交付形态。
- 训练页先讲配置，再讲指标，避免直接堆表。
- 指标页建议用大数字：Box mAP50 0.99480、Pose mAP50 0.99482、Pose mAP50-95 0.99461。

### 第 11-14 页：落地能力和未来价值

- 部署页强调同一套 Web/数据库逻辑支持 PC 和 RDK。
- 项目亮点页使用短句和图标卡片，适合答辩快速讲述。
- 风险页要主动说明低照度、反光、网络抖动和误识别处理思路。
- 展望页把技术演示提升到真实车库地图、移动端寻车和长期运维。

## 5. 候选视觉风格

### A. 深蓝霓虹工程风

- 关键词：地下车库、青蓝光效、数据流、边缘计算板、Web 控制台。
- 适合：技术答辩、项目展示、强调 AI 工程实现。
- 优点：和项目题材最贴合，视觉冲击强。
- 风险：如果光效过多，文字页需要控制透明面板和对比度。

### B. 白底理性竞赛风

- 关键词：浅色背景、蓝绿线框、信息图、工程图纸、清晰表格。
- 适合：课程答辩、论文式汇报、评委需要快速读清内容。
- 优点：可读性强，训练结果和架构图清楚。
- 风险：科技感相对弱，需要靠局部三维图和色块增强记忆点。

### C. 投资路演黑金风

- 关键词：黑色背景、金色重点数字、大标题、少文字、强故事感。
- 适合：短时间路演、成果展示、强调项目价值和落地前景。
- 优点：气场强，封面和结尾页很抓人。
- 风险：技术细节页需要避免过度营销化。

### D. 3D 沙盘全息风

- 关键词：全息车库沙盘、路线光轨、浮层 UI、模型管线、立体图标。
- 适合：展示“实现效果”和“未来落地场景”。
- 优点：直观、沉浸，适合做封面和系统总览。
- 风险：需要约束每页信息密度，否则容易像海报而不是 PPT。

## 6. 推荐默认方案

如果没有额外偏好，建议选 A + D 混合：封面、方案总览和展望使用 3D/霓虹场景增强吸引力；技术页使用深蓝底 + 清晰框图 + 高对比浅色文本，保证讲解时可读。

## 7. 后续制作原则

- 每页只讲一个核心结论。
- 正文字号不低于 22 pt，关键数字不低于 48 pt。
- 架构和线程图优先使用自绘矢量/高清 PNG，不直接截图长文档。
- image2 适合生成封面、章节页、场景图和风格底图；技术流程、指标表和代码事实用本地绘图或 PPT 原生形状保证准确。
- 正式 PPT 文件建议输出为 `.pptx`，并保留可复现生成脚本。
